# 正しい指導

## シーン構造

```
価値変化アーク:
[最初のコードレビュー] ──[月をまたぐ積み重ね]──→ [1on1の言葉] ──[「その違いはあなたにしか分からない」]──→ [未処理の違和感]

主人公: (正確な指導という確信) → (正しさの反復) → (違和感の抑圧)
A男:    (承認の欲求) → (表情の変化) → (言葉の中に収納された告発)
```

| McKee要素 | 内容 |
|-----------|------|
| 価値軸 | 正確さ・指導 ⇄ 加害・抑圧 |
| シーン機能 | 主人公の「正しい言葉」が積み重なって暴力になる過程を、月をまたいで圧縮描写する。ACT1/02でA男が主人公を攻撃したように、今度は主人公がA男を攻撃している構造的対称性を示す |
| 価値変化 | ニュートラル（指導関係の開始）→ マイナス（言葉の蓄積・A男の表情変化）→ マイナスの深化（「その違いはあなたにしか分からない」という告発） |
| 主要モジュール | `19_scene-mastery`, `23_character_components`, `14_causal-chain`, `22_cast-design`, `05_controlling-concept`, `21_character-dimension` |

## 一　コードレビュー

　九月の終わりに、A男の書いたコードを初めてレビューした。

　Pull Requestを開くと、差分が四百行あった。
　主人公は上から読んだ。
　百行目で止まった。

> **[19_scene-mastery]** ✅ 「百行目で止まった」——行数という具体的な数字が、主人公の読み方の精密さを示す。「止まった」という動詞が、問題の発見を視覚的に表す。

　変数名が `tmp` だった。
　その変数は、フォームから受け取ったユーザーIDを格納していた。
　`tmp` という名前は、何の情報も持っていなかった。

　主人公はコメントを書いた。

　「この変数の定義を教えてください。」
　「`tmp` という命名は意図が不明です。`receivedUserId` 等、意味を持つ名前が適切です。」
　「また、この変数はスコープ外でも参照される可能性がありますか？」

> **[23_character_components]** ✅ コメントの文面——三つの問いが全て論理的に正当。主人公の言葉に一点の瑕疵もない。しかしその正確さが、読み手（A男）にとって何を意味するかは書かれていない。行動の描写だけで、残酷さが立ち上がる。

　翌日、A男から返信が来た。
　「修正します」とだけ書いてあった。

　修正されたコードを確認した。
　変数名は変わっていた。
　ただ、別の箇所に同じ問題があった。

　主人公はまた止まった。
　また、コメントを書いた。

> **[14_causal-chain]** ✅ 「また止まった。また、コメントを書いた。」——反復の描写が、一回ではなく継続する構造であることを予告する。「また」の二回の使用が、パターンの形成を示す。

## 二　十月の1on1

　月次の1on1を十月から始めた。

　三十分の枠で、主人公とA男が会議室に入る。
　A男はいつも先に来ていた。
　ノートを開いて待っていた。

　進捗の確認から始めた。
　A男は答えた。
　声が少し低かった。

> **[22_cast-design]** ✅ 「声が少し低かった」——A男の状態変化の初期兆候が音として示される。主人公はこの変化を観察している。しかし何も処理しない。観察と処理の分離が、主人公の機能の特性として示される。

　「先月の工数見積もりが実際の三倍でした」と主人公は言った。
　「原因を教えてください」

　A男が少し間を置いた。
　「仕様の変更が後半に入って」と言いかけた。

　「仕様変更の可能性は最初から考慮できましたか」

　「……できたかもしれないです」

　「感覚で動いてから工数を嘆くのは逆順です。」
　「見積もりは動き始める前に行うものです。」

> **[05_controlling-concept]** ✅ 「感覚で動いてから工数を嘆くのは逆順です」——主人公の言葉は正しい。しかしこの正しさが、A男が「感覚で動いた」という前提を批判している。A男の動き方を「逆順」と定義することで、A男のやり方を根本から否定している。主人公はそれを指摘していると思っていない。

　A男はノートに何か書いた。
　何を書いたか、主人公からは見えなかった。
　見ようとも思わなかった。

> **[20_interpretation-layer]** ✅ 「見ようとも思わなかった」——主人公の内部が一文で示される。関心の欠如ではなく、「そこを見る必要がない」という判断。しかし読者には、A男のノートに何が書かれているかへの関心が生まれる。

## 三　十一月の1on1

　十一月の1on1の前に、A男からSlackメッセージが来た。
　「今週は体調が少し良くなかったです。作業が遅れました」という内容だった。

　主人公は返信を書いた。
　「了解です。1on1で確認します」

　会議室に入ると、A男はまたノートを開いて待っていた。
　今月は、顔が少し白かった。

> **[17_symbolism-and-image]** ✅ 「顔が少し白かった」——A男の身体状態が視覚的に示される。主人公はこれを観察している。後に「見ている」が「処理しない」という構造の最初の確立。白という色が、ACT5の「ホワイトボード」と「白いページ」の予型として機能する。

　「先月の件について」と主人公は始めた。
　「強制的にやり方を変えさせたいわけではありません」
　「ただ、論理的な手順として、見積もりを先に行う方が効率が高いです」
　「これは強制ではなく、論理的な要請です」

　A男が言った。
　「……その違いは、あなたにしか分からないと思います」

> **[21_character-dimension]** ✅ 「その違いはあなたにしか分からない」——このシーンで最も重い言葉。A男が「強制と論理的要請の違い」を主人公だけが知っていると言っている。言い換えれば、「受け取る側にはその違いが届いていない」という告発。主人公はこの言葉の意味を処理しない。

　主人公は少し考えた。
　A男の言っていることは正しかった。
　強制かどうかは、意図で決まる。
　主人公の意図は強制ではない。
　だから、強制ではない。

　「その通りです」と主人公は言った。
　「だから論理的な要請と言っています」

> **[05_controlling-concept]** ✅ 「だから論理的な要請と言っています」——主人公がA男の言葉を「同意」として受け取っている。しかしA男の言葉は同意ではなかった。意図と影響の乖離が、この応答で完成する。主人公は自分の意図だけを見て、A男への影響を見ていない。

　A男は何も言わなかった。
　ノートに、また何か書いた。

　1on1の残り十分で、タスクの確認をした。
　A男は全て「了解しました」と答えた。

　会議室を出て、主人公は自席に戻った。
　作業ログを開いた。
　A男のパフォーマンスについての記録を更新した。

　「工数管理の改善が必要。論理的な手順を再確認中」と書いた。

> **[14_causal-chain]** ✅ 「論理的な手順を再確認中」という記録——主人公の視点ではこれが正確な状態記述に見える。しかし「再確認中」という進行形の語が、問題をA男の側に帰属させている。加害が記録の形式で更新されていく構造。

　A男の顔が少し白かったことは、書かなかった。

　気になったが、健康管理は個人の問題だと思ったからだ。

> **[22_cast-design]** ✅ 「気になったが、健康管理は個人の問題だ」——主人公が関心を持ちながら、その関心を「個人の問題」という定義で閉じている。「気になった」という感情の萌芽が存在することを認めながら、処理の回路がそれを遮断する。この閉じ方が、ACT1のA男の攻撃についてノートを書いた時と構造的に同じであることを、読者は気づく。
