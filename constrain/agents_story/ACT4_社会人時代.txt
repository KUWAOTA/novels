Constrain — ACT4 シーン描写例（20本）
時代：Google Japan 社会人時代
主人公：善意の教育が加害に変わっていく / 鏡像認識
A：再び主人公の下に入る / 教育の中で過去を再演される

凡例: 【場】=舞台・時間、【価値変化】=このシーンの価値の動き

================================================================================

S4-01「Google Japanの初日」
【場】Google Japan オフィス / 4月 / 入社初日
【価値変化】使命感 → 適応の確認

受付を通ってIDバッジを受け取った。オープンスペースのデスクにPCが並んでいた。英語の会話、コードの議論、ホワイトボードの数式。ここでは論理が通貨だった。主人公は深呼吸した。環境検証：合格。今日から本番だ。——その日は、同じフロアにA男がいることに気づかなかった。

================================================================================

S4-02「A男の姿」
【場】Google Japan ランチルーム / 入社後一週間
【価値変化】日常 → 布石の設置

トレーを持って席を探していたら、見覚えのある背中があった。一瞬、記憶の索引が動いた。中学、あの廊下、あの倉庫の影。A男だった。同じ会社にいる。チームは違う部門だった。向こうも気づいているはずだが、何も言わなかった。主人公も何も言わなかった。論理的に、すべきことが思いつかなかったから。

================================================================================

S4-03「再会のシーン」
【場】廊下の角 / 入社後一ヶ月
【価値変化】過去の再浮上 / 感情処理の不能

「久しぶり」とA男が言った。声に感情の起伏がなかった。「同じ会社でしたね」と主人公は言った。「まあね」。それだけだった。握手もなかった。二人は逆方向に歩いた。主人公はその夜、ノートに「A男との再会。定性的な感情は不明。定量的な変化なし」と書いた。でも眠りが少しだけ浅かった。

================================================================================

S4-04「昇進の発表」
【場】チームの定例会 / 入社3年目
【価値変化】正当な報酬 / しかし構図が始まる

「主任エンジニアとして昇進してもらいます」とマネージャーが言った。主人公は「分かりました」と言った。この評価は正しい、という確信があった。同時に、A男がまだ同じポジションにいることも、その日から数値的な記録として頭に入った。

================================================================================

S4-05「A男が部下に」
【場】組織変更後の初チームミーティング
【価値変化】使命感 → 純粋な善意の始まり→悲劇の種

部門再編でA男が自分のチームに入った。主人公は最初、過去の感情を持ち込まないように整理した。A男は技術者として能力がある。ただ、曖昧な仕様で進みがちな癖がある。「俺が正しく導けば、A男はもっと良くなる」と思った。——その考えが、どこからくるものかは、まだ分からなかった。

================================================================================

S4-06「最初のコードレビュー」
【場】ミーティングルーム / 業務
【価値変化】善意の指導 → 初めての亀裂

A男のプルリクエストを見た。変数名が曖昧で、コメントが感覚的だった。「この変数名の定義は？」と主人公は聞いた。「まあ分かるかな、と思って」とA男は言った。「『分かる』の根拠がないと、二週間後の自分も分からなくなります。厳密にしてください」。A男は「…はい」と言った。その声のトーンを、主人公は気にしなかった。

================================================================================

S4-07「工数の指摘」
【場】1on1ミーティング / 翌月
【価値変化】善意の教育 → 加害の萌芽（GF-003対応）

A男が「このタスクは手動確認の工数が多くて」と説明した。主人公は言った。「だから最初に自動化の設計をすべきでした。感覚で動いてから工数を嘆くのは逆順です」。正しい。完全に正しい。でもA男の顔が微妙に変わったのを、主人公は見ていた——見ていたが、処理しなかった。感情的なデータより論理的な改善の方が、先だと思ったから。

================================================================================

S4-08「定義の応酬」
【場】打ち合わせ / 昼
【価値変化】善意の説明 → 相手の限界点に触れる（GF-001対応）

「この仕様書、ここの前提が曖昧です」と主人公は言った。「だから動く前に定義を固めないといけない。これは職業人として当然の責任です」。A男は「……でも、あなたがそれを言うのは、強制に近いと思う」と静かに言った。主人公は一瞬止まった。「強制ではなく論理的な要請です」。「その違いは、あなたにしか分からないと思う」。——会話はそこで終わった。A男は席に戻った。

================================================================================

S4-09「A男が泣く」
【場】ミーティングルーム / 深夜の残業
【価値変化】正義の貫徹 → 加害の発現（GF-002対応）

二人きりになったミーティングルームで、主人公がレビューコメントを一つずつ説明していた。「この部分の論理の飛躍の理由を教えてください」「この前提はいつ合意されましたか」「なぜこの選択肢を取ったのか」。A男が黙った。しばらく後、声が詰まった。目が赤くなっていた。主人公は止まった。「……今の話のどこで感情的になりましたか」と聞いた。A男は何も言わなかった。

================================================================================

S4-10「なぜ泣くのかが分からない」
【場】自室 / 帰宅後
【価値変化】善意の確信 → 最初の疑念

ノートに書いた。「A男が泣いた。論理的な指摘に対する感情的反応。原因不明。感情負荷の計算が足りなかったか？」。でもすぐにその疑念を消した。「感情的であることは彼の問題。俺はやるべきことをしただけだ」。消した。でも消えなかった。

================================================================================

S4-11「Constrainとの矛盾」
【場】深夜のオフィス / Constrainのコードを眺める
【価値変化】使命感の内部崩壊の始まり

Constrainのリポジトリを開いた。「曖昧な表現を検出して警告する」ツール。今日A男にやったことと同じだと、突然思った。Constrainはコードにエラーを出す。自分はA男に「エラー」を出し続けている。コンパイラがコードにエラーを出したら、コードは修正されるか、コンパイルを諦めるか、どちらかしかない。A男は、どちらに向かっているのか。

================================================================================

S4-12「A男の欠勤が始まる」
【場】月曜日の朝 / 空席
【価値変化】日常の変化 / 主人公の誤認

A男の席が空だった。「体調不良」とSlackに一行あった。珍しくない。でも翌週も、翌々週も続いた。主人公はマネージャーに「A男の業務品質に問題があります。教育的サポートが必要では」と言った。——それがすでに、何かを見誤っていることに気づかなかった。

================================================================================

S4-13「電話での指摘」
【場】主人公の自室 / 深夜 / A男の自宅へ電話
【価値変化】善意の教育 → 追い詰める加害（最大化）

「業務の進め方について整理したい」と電話した。A男は出た。声が静かだった。「あのタスクの設計の問題点を説明します」。A男は「……今でなくてもいいですか」と言った。「課題は先送りにしても消えません。今の方が効率的です」。A男は何も言わなかった。でも電話を切らなかった。ただ、沈黙があった。——主人公が気づかなかったのは、沈黙にも意味があるということだった。

================================================================================

S4-14「A男の家族からの連絡」
【場】主人公のスマホ / ある午後
【価値変化】日常 → 現実の着地

見知らぬ番号から着信があった。「弟の件でご連絡しました」。A男の姉だった。「弟が出社できない状態になっています。また連絡があれば困るので、ご配慮をお願いします」。主人公はしばらく画面を見た。「配慮」という言葉の定義が、この文脈では分からなかった。でも何かが、内側で音を立てた。

================================================================================

S4-15「主人公の夜」
【場】自室 / 深夜
【価値変化】自覚の前夜

B子のことを久しぶりに考えた。B子に「論理の矛盾」を指摘していた日々を思い出した。その時B子は「そうか、ごめんね」と言っていた。B子と、A男が重なった。でも違う。B子は自分から泣いたのではなく——いや、B子は泣かなかった。だから気づかなかったのか? B子も本当は——。その仮説を、主人公は最後まで書けなかった。

================================================================================

S4-16「鏡像認識の瞬間」
【場】洗面台の前 / 早朝
【価値変化】加害（無自覚） → 加害（自覚） / 否定の否定の完成

鏡の中の自分を見た。なぜA男が自分を嫌ったのか、小学校の時から。A男の父親が「なぜ100点が取れない」と言い続けた。それがA男を壊した。自分はA男に「なぜ定義が曖昧なのか」と言い続けた。言い方は違う。目的も違う。でも——構造が、同じだ。A男にとって、自分は「また別の父親」だったのだ。主人公は水を出したまま、しばらく動けなかった。

================================================================================

S4-17「Constrainが評価される記事」
【場】Webの記事 / 同じ朝
【価値変化】皮肉な価値の逆転

スマホのニュースフィードに、Constrainを取り上げた記事が出ていた。「曖昧さを一掃するHaskell製文書検証ツール」。コメント欄に「これ使うと文章が全部エラーになって逆に怖い」とあった。別のコメントに「正確さって、使い方によっては人を不安にさせるよな」とあった。主人公はその言葉を三回読んだ。

================================================================================

S4-18「A男の席が空になる日」
【場】オフィス / ある月曜日
【価値変化】喪失の再来

A男の席からパソコンが消えていた。引き出しが空だった。人事から「A男さんは休職扱いになりました」とだけ連絡が来た。主人公はメールを読んで、返信をしなかった。B子の席が空になった朝のことを思い出した。あの時も、こういう感じだった。「結局また、救えなかった」という言葉が、頭の中で鳴った。

================================================================================

S4-19「ミーティングルームで一人」
【場】深夜のオフィス
【価値変化】自責 / 誤信念の最終的な崩壊前夜

A男が使っていたホワイトボードがまだ消されていなかった。フローチャートの途中で止まっていた。A男の字で。主人公は消しゴムを持ったが、止めた。フローチャートに何かが書いてある。仕様の案だと思ったが、よく見ると設計ではなく、「Aの手順」「Bなら次へ」という、何かの判断フローチャートだった。読んでいくと、それが「生きることの条件分岐」だと分かった。

================================================================================

S4-20「一通のメッセージ」
【場】主人公のスマホ / 深夜
【価値変化】ACT5への橋

A男からメッセージが来た。短かった。

「明日、話したいことがある。あの場所に来てほしい」

送信場所のピンが付いていた。橋の上だった。中学の帰り道、B子と歩いた橋だったかどうかは分からない。でも主人公は三秒見てから、ジャケットを着た。
