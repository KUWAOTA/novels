# setting_idea 採用判断フレーム

## 目的

`setting_idea` 配下の案を、「設定がある」ではなく「本文で何を成立させるか」で判定するための基準。

このフレームに合わない案は削除するか、意味が立つまで修正する。

## 採用条件

各案は最低でも次の3点を満たすこと。

1. 著者の5軸のどれかを強める
2. `story-compiler` のどれかの要件に接続できる
3. シーン、関係、誤信念、転換のいずれかを前に進める

## 著者の5軸

1. リアルな阻害の描写
2. 苦しみの描写
3. 論理や仮説で乗り越えていく
4. クレバーで不器用なリアリティ
5. 傷を知ったうえで世界の曖昧さを能動的に肯定する

## 主な story-compiler 参照先

- `05_controlling-concept`: 統制概念、対立概念、価値変化
- `06_protagonist`: 欲求、善の中心、心性、圧力下の露出
- `17_symbolism-and-image`: 反復する場所、モチーフ、イメージの回収
- `19_scene-mastery`: 価値が動くシーンか
- `21_character-dimension`: 矛盾があるか、圧力で露出するか
- `22_cast-design`: 鏡像、対照、関係機能
- `24_discernment`: 単純化、説明臭さ、作者都合の排除

## 削除対象

以下に当てはまる案は削除候補。

- 風景や癖の説明だけで、因果や価値変化に寄与しない
- 「それっぽい」だけで、誰のどの苦しみに効くか不明
- テーマを説明するだけで、行為や場面に落ちない
- 悪役化、天使化、診断語依存など、単純化が強い
- 既存案の言い換えで、新しい圧力や差異を生まない

## 修正の仕方

意味が弱い案は、次のどれかに変換する。

- ディテール案 -> シーンで圧力がかかった時に露出する行為へ変換
- 設定案 -> 誤信念の起点か、終盤の転換伏線へ変換
- 雰囲気案 -> 象徴の反復か、身体感覚の記述へ変換
- 説明案 -> 誰かの誤読・失敗・未達として現れる場面へ変換

## 運用メモ

- 新規案は「タイトル / 内容 / 即する軸 / SC / 効用」の5点セットで書く
- 無理に全件残さない
- 迷ったら、本文で一番使い回せる案だけを残す
