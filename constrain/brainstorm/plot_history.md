## 1. 物語の核心テーマ（変遷）

### v1 — 初期テーマ（00_core-theme.yaml）

```yaml
core_theme:
  one_word: 言語化
  description: >
    言葉の厳密さ（well-defined）と曖昧さ（ill-defined）の
    二面性を通じて、人と人がわかり合うこと・世界を広げることの
    本質に迫る物語。

  deeper_layers:
    - 再現性のある言語は人を救うが、それだけでは世界は広がらない
    - 厳密さは防御、曖昧さは冒険――両方が必要
    - 「分かる」ことと「触れる」ことは別の営みである
    - 言葉は人と人を繋ぐ架け橋（再現性→共感→創造という段階がある）
```

### v2 — 歩き方のメタファー（value_map v3）

```
「歩き方は一つではない」
・主人公の足 = 義足（精度・言語）
・いじめっ子の足 = 素足（空気を読む力）
・主人公の過ち：「義足の方が優れている」と信じ他人の素足を切って義足を強制した
・着地点：歩き方は足でも義足でもいい。他人の足を切らなくてもいい。
```

### v3 — 現在採用（girlfriend.md 最終版）

```
「厳密な理解を必要とする人間が、不完全な社会と共存する方法を見つける話」

テーマ一文：
  厳密さで世界を守ろうとした人間が
  誤解と傷でできた社会を
  それでも愛せるようになる話

価値変化弧：
  理解すれば救える
  ↓
  理解しても救えない
  ↓
  理解できなくても寄り添える

キーワード：
  絶望を理解した後の肯定（希望の物語ではない）
  制御 → 共存
  言語 → 身体（クライマックス）
```

### 価値軸候補（00_core-theme.yaml）

```
primary:
  axis: 厳密さ ←→ 曖昧さ
  positive: 曖昧さを含む言語による創造・抱擁・世界の拡張
  negative: 厳密さへの固執による世界の縮小・孤立

secondary:
  - axis: 孤立 ←→ 繋がり
  - axis: 防御 ←→ 冒険（自分を守るための生き方 vs 世界を旅する生き方）
  - axis: 理解 ←→ 抱擁（頭で分かることと心で受け入れることの違い）
```

  → テーマ価値：正義と言語の暴力性、命令と依頼の構文差

- **RM-002「解釈の決めつけと言語運用レベルの差」**（2026-02-24）  
  彼女「結局母親の話だから感情的になったんでしょ」→ 主人公の自己認識と不一致  
  処理能力や誠実さの問題ではなく「抽象度の違い」が存在する  
  → 主人公が「正しさ」を突きつけることで相手を追い詰める繰り返しパターン

- **RM-003「最適化は暴力になるか」**（2026-02-24）  
  彼女「歩み寄りたいと思って言ったのに、『工数を最小化したい』って言われたのが嫌だった」  
  関係調整モード vs 最適化モード の衝突  
  → テーマ価値：最適化は暴力になるか、愛と効率の衝突
